cocoapod各种问题的解决方案

pod install 报警告（debug、release。。）

解决方案：
other link flags : $(inherited)
删除pod，重新pod install
clean proj

pod中添加第三方库之后，ld错误：

解决方案：
Build archive architecture only: debug:yes
pod第三方库报错，duplicate

解决方案例如：
import “YYImage.h” 改为 #import <YYImage/YYImage.h>

最后找不到pod工程库：

解决方案：
找不到framework错误，手动+入

pod更新错误或超时，解决方案：

git config http.postBuffer 524288000
git config https.postBuffer 524288000
git config –global http.postBuffer 1048576000
git config –global http.lowSpeedLimit 0
git config –global http.lowSpeedTime 999999

pod 添加私有库pod地址

pod repo add repositories http://xxx.168.xxx.xxx:8081/lichanghong/repositories.git
前提需要先创建 git@gitlab.xxx.com:lichanghong/repositories.git库，不可为空

私有库验证正确性
pod spec lint ThirdMediator.podspec –verbose –allow-warnings –sources=’git@gitlab.xxx.com:lichanghong/repositories.git’
推到私有pod仓库
pod repo push Repositories ThirdMediator.podspec –verbose –allow-warnings –sources=’git@gitlab.xxx.com:lichanghong/repositories.git’

更新私有仓库

pod repo update Repositories

pod库问题总结

能search到库，还是报错：[!] Unable to find a specification for BaseUtil_UI (~> 0.1.1) depended upon by BaseUI
pod repo update 不行
解决方案：在podfile里加source地址，地址指向私有库！

静态库问题

[!] The ‘Pods-BusUtil_Example’ target has transitive dependencies that include statically linked binaries: (ThirdMediator)
s.static_framework = true

冲突问题

[!] [Xcodeproj] Generated duplicate UUIDs:
s.dependency ‘xxx/Frame’
s.dependency ‘xxx/Net’
类似的这种引入方式导致的，需要直接引入 s.dependency ‘xxx‘

需要gitignore

Example/HomeCells.xcworkspace/xcuserdata/lichanghong.xcuserdatad/UserInterfaceState.xcuserstate

找不到依赖

  • ERROR | [iOS] unknown: Encountered an unknown error (Unable to find a specification for BaseUI depended upon by OneViewController BaseUI在私有库里，也能搜索到，但是却在被依赖并pod repo push的时候报错
    解决：”对BaseUI进行升级之后没有pod repo push上去，先升级，然后再被依赖” 然并不是这个原因！ OneViewController没有打tag导致的,打tag上传，还是报错如下：
    [OneViewController/VC] xcodebuild: Returned an unsuccessful exit code.
    升级pod也不行
    最后指定sources： pod lib lint OneViewController.podspec –verbose –allow-warnings –sources=’git@gitlab.xxx.com:lichanghong/kdrepositories.git,https://github.com/CocoaPods/Specs'
    有时候指定source也不行，还是报错找不到module，解决方法是把#import “” 改为 #import <A/A.h>

not added the source repo that hosts the Podspec to your Podfile.

网络上的解决方案：未验证 http://www.mengyueping.com/2018/08/16/iOS_CocoaPods_03/

The ‘Pods’ target has transitive dependencies

  • 原因：如果私有库添加了静态库或者 dependency 用了静态库
  • 解决：那么执行 pod spec lint 或者 pod repo push 时候需要加上 —user-libraries 选项。否则会出现 The ‘Pods’ target has transitive dependencies 错误。
    或使用终极方式：spec.user_target_xcconfig = { ‘CLANG_ALLOW_NON_MODULAR_INCLUDES_IN_FRAMEWORK_MODULES’ => ‘YES’ }。

取消pod库里的警告：inhibit_all_warnings!

可以配置执行脚本：

:execution_position 选项有 [:before_compile, :after_compile, :any]

s.script_phase = { :name => “Script Name”, :script => “echo ‘Hello World’”, :execution_position => :any , :shell_path => “/bin/sh”}

c++库的错误

Undefined symbols for architecture i386:
“operator new[](unsigned long)”, referenced from:
_MD5CPP in libETString.a(md5.o)
添加： s.libraries = ‘c++’
Undefined symbols for architecture armv7: cocoaPods
PodFile 注释掉：#use_frameworks!

纯OC项目遇到swift报错

- ERROR | xcodebuild:  MRefresh/MRefresh/Classes/MRefresh/UIScrollViewExtension.swift:51:64: error: argument of '#selector' refers to instance method 'gestureRecognizerUpdated' that is not exposed to Objective-C

Vzinspector 在封装为pod之后，外部再次对参数封装报错：

Returning block that lives on the local stack
有俩block报错，解决方式：把这个mm类的-fno-objc-arc去掉
运行起来之后崩溃：podspec有问题，升级，有些.mm文件应该是arc的，自动install之后是错误的

Pod repo update 总是需要用户名密码

解决方案：重启电脑！
